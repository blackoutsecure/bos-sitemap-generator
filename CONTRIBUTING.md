# Contributing to Blackout Secure Sitemap Generator

Thank you for your interest in contributing! We welcome contributions from the community.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/bos-sitemap-generator.git`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Install dependencies: `npm install`

## Development

### Build

```bash
npm run build
```

### Testing

```bash
npm test
npm run test:watch
npm run coverage
```

### Linting & Formatting

```bash
npm run lint
npm run format
npm run check  # Run all checks
```

## Making Changes

1. Create a descriptive commit message
2. Include tests for new functionality
3. Ensure all tests pass: `npm test`
4. Update documentation as needed
5. Follow the existing code style

## Pull Request Process

1. Update the README.md with any new features or changes to usage
2. Ensure all tests pass: `npm run check`
3. Create a pull request with a clear description of changes
4. Link any related issues

## Code Style

- Use ESLint configuration provided
- Use Prettier for formatting
- Follow existing patterns in the codebase
- Write clear, descriptive comments

## Testing Guidelines

- Write unit tests for new functions
- Add integration tests for new features
- Ensure edge cases are covered
- Test with multiple Node versions if applicable

## Reporting Issues

- Use GitHub Issues for bug reports
- Include detailed reproduction steps
- Provide environment information (OS, Node version, etc.)
- Include relevant error messages and logs

## Feature Requests

- Discuss significant features in an issue first
- Describe the use case and expected behavior
- Consider backward compatibility implications

## Questions?

- Open a GitHub Discussion
- Check existing issues for similar questions
- Review documentation and examples
